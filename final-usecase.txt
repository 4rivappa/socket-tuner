### Entire flow
RL environment (inference)<-> (talks to cluster and gets values) <-> EKS cluster

Final State of the system after training:
In EKS Kubernetes cluster, we will be hosting few nodes.
Each node contains this eBPF agent.
eBPF agent will be responsible to load and observe eBPF programs and data from programs.
eBPF program will be able to couple of things
- observe closed TCP connections and send metrics to the eBPF agent.
- intercepts each socket open connection TCP, if there is a socket configuration present for that remote IP - it will apply it, else it changes nothing returns normal
Predictions for each remote IP is loaded by eBPF agent into eBPF maps, prediction will be from a small LLM expert
